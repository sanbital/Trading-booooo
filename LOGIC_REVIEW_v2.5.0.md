# v2.5.0 로직 리뷰 — 과거 추세 × 현재 호가·체결

## 1. 기존 구조에서 확인한 핵심 문제

기존 엔진도 기간 추세와 동적 호가를 모두 수집했지만, 최종 단계에서는 두 결과를 점수로 합산하는 비중이 컸습니다. 이 때문에 다음 문제가 발생할 수 있었습니다.

- 스캔 초기에 받은 24시간 티커 가격으로 진입가를 계산해 WebSocket 종료 시점 가격과 어긋날 수 있음
- 동적 표본 부족인데도 위험 패턴이 없다는 이유로 호가 안전성 게이트가 PASS될 수 있음
- 목표가와 손절가가 실제 지지·저항 없이 ATR만으로 생성될 수 있음
- 현재가 진입 계획을 눌림 가격에 그대로 재사용해 대기 손익비가 부정확해질 수 있음
- RSI만으로는 EMA 이격·누적 급등·ATR 대비 과속을 충분히 차단하지 못함
- 거래대금과 스프레드는 보지만 실제 투입금이 상위 호가를 얼마나 밀어내는지 확인하지 못함
- 일봉·4시간의 정렬 상태를 상승/하락 신호 하나로 단순화해 회복·눌림·혼조가 섞임

## 2. 수정한 판정 구조

### A. 과거 캔들: 방향과 구조 결정

각 시간축을 다음 다섯 상태로 분류합니다.

- `FULL_BULL`: 가격 > EMA9 > EMA21 > EMA50, EMA21 기울기 양수
- `BULL_PULLBACK`: 중기 상승 구조 안에서 가격이 EMA9·EMA21 부근으로 눌림
- `RECOVERY`: 장기 저항 아래에서 단기 EMA가 회복 중
- `MIXED`: 방향이 불명확하거나 정렬이 충돌
- `BEAR`: 하락 정렬 또는 중기 추세 훼손

과열은 RSI 단독이 아니라 다음을 합산합니다.

- RSI 초과
- 가격과 EMA21의 이격
- 최근 12봉 수익률의 ATR 대비 크기
- 거래대금 급증
- 일봉 누적 급등과 24시간 급등

### B. 실시간 호가·체결: 지금 실행 가능한지 결정

WebSocket 관찰 종료 시점의 마지막 호가를 기준으로 다음을 계산합니다.

- 최신 최우선 매수·매도호가 중간가
- 기준가격의 나이
- 동적 호가 관찰시간·서로 다른 호가 수·동시간대 체결 수
- 초반·중반·확인 3구간 표본 분산
- 실제 체결량으로 설명되지 않는 대형 매수벽 취소
- 매도벽의 체결 후 재보충·흡수
- 매수벽 소진 후 지지 붕괴
- 매도벽 실체결 소진 후 지지 전환
- 상위 호가 양방향 가시잔량
- 예상 투입금 기준 매수 VWAP 슬리피지

표본이 부족하면 `특이 위험 없음`이 아니라 `판정 불가`로 처리하고 BUY를 금지합니다.

### C. 과거와 현재의 결합: 실제 가격 구조 생성

과거와 현재는 단순히 점수만 섞지 않습니다.

1. 과거 15분·4시간 EMA와 스윙 저점에서 지지 후보를 찾음
2. 실시간 표본이 충분하고 안전할 때만 반복 매수벽·돌파 지지를 추가함
3. 진입 실행가 아래에서 가장 가까운 유효 지지를 구조적 지지로 선택함
4. 과거 스윙 고점·일봉 고점과 실시간 반복 매도벽에서 실제 저항을 찾음
5. 진입 실행가 위에서 가장 가까운 저항을 목표 상한으로 선택함
6. 손절은 구조적 지지 아래에 ATR·틱 완충을 두고 계산함
7. 목표는 실제 저항 앞에서 청산되도록 완충하고 ATR 상한도 적용함
8. 수수료와 진입·청산 슬리피지를 반영한 순손익비를 계산함

구조적 지지와 구조적 저항 중 하나라도 없으면 현재가 BUY가 불가능합니다.

## 3. 최종 게이트

현재가 BUY는 다음을 모두 통과해야 합니다.

- 기간 데이터 완성
- 시장경보 없음
- 최소 거래대금 충족
- 최근 체결 최신성
- 10초 이내 최신 실시간 호가 사용
- 15분 상승·눌림·회복 구조
- 4시간 상승·눌림·회복 구조
- 일봉 하락 정렬 아님
- 복합 과열 기준 통과
- 동적 호가·체결 표본 충족
- 스프레드 35bp 이하
- 예상 투입금 대비 양방향 호가 깊이 1.5배 이상
- 예상 매수 슬리피지 20bp 이하
- 초단기 체결·호가 수급 악화 없음
- 동적 호가 위험 패턴 없음
- 실제 구조적 지지·저항 존재
- 비용 포함 손절폭 범위 충족
- 비용 포함 손익비 기준 충족
- 종합점수 기준 충족

점수는 후보 정렬용이며, 하나의 강제 게이트도 점수로 상쇄할 수 없습니다.

## 4. 눌림 대기 로직

현재가가 부적합해 WAIT가 되면 현재 계획을 재사용하지 않습니다.

- 현재가 아래의 EMA·스윙 지지·안전한 실시간 지지로 눌림 구간 생성
- 눌림 상단을 가상 진입가격으로 설정
- 해당 가격에서 구조적 지지·저항·손절·목표·비용 포함 손익비를 새로 계산
- 손익비 또는 손절폭이 기준에 미달하면 대기 매수가도 미제시
- 가격 도달 후 15분봉 회복과 최신 동적 호가를 다시 확인하도록 `RECHECK_REQUIRED` 처리

## 5. 보유기간 처리

백테스트 근거 없이 일괄적으로 2~6주를 제시하지 않습니다.

- 기본 관찰: 1~12시간
- 정렬·변동성 양호: 6~48시간
- 15분·4시간 완전 상승: 2~10일
- 충분한 장기 데이터와 낮은 과열까지 충족: 5~20일

보유기간은 확정 예측이 아니라 시간 손절과 재검토 주기를 정하는 보조 정보입니다.

## 6. 검증 범위

- TypeScript 정적 검사 통과
- 프론트엔드 JavaScript 문법 검사 통과
- 엔진·통합선정·백테스트 단위 테스트 45개 통과
- 동적 표본 부족, 오래된 티커와 최신 호가 불일치, 일봉 급등 과열, 얕은 호가 깊이 차단 시나리오 추가

실거래소의 향후 시세를 사용한 성과 검증이나 Supabase 실배포는 포함하지 않았습니다. 실시간 동적 판정 성능은 운영 중 페이퍼 로그를 누적해 전진검증해야 합니다.

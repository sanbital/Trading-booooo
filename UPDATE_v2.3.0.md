# v2.3.0 업데이트 — 동적 호가 60~90초 관찰

기존 v2.2.0 설치본을 GitHub 웹에서 덮어쓰는 업데이트입니다. VS Code와 터미널은 필요하지 않습니다.

## 변경 내용

- 최종 후보 8개를 한 WebSocket에서 동시에 기본 60초 관찰
- 거래대금 경계 후보가 있으면 90초 자동 연장
- BUY 최소 동적 표본: 관찰 45초·서로 다른 호가 25회·동시간대 체결 20건
- 초반·중반·확인 3개 구간마다 호가와 체결 표본이 분산됐는지 검증
- 구간 표본 미달 시 점수가 높아도 BUY 및 조건부 눌림 매수가 차단
- 카드와 복사 리포트에 `구간 3/3`과 구간별 호가·체결 수 표시
- 브라우저 분석 요청 제한시간 최소 220초

## GitHub 적용 순서

1. `Trading-booooo-v2.3.0-GitHub-Update.zip`을 다운로드하고 압축을 풉니다.
2. GitHub `Trading-booooo` → `Code` → `Add file` → `Upload files`로 이동합니다.
3. 압축을 푼 폴더 안의 모든 항목을 업로드 영역에 끌어놓습니다.
4. Commit message에 `Extend dynamic orderflow observation v2.3.0`을 입력합니다.
5. `Commit changes`를 누릅니다.
6. `Actions`에서 Supabase 배포와 Pages 배포가 모두 초록색인지 확인합니다.
7. 기존 개인 URL에서 `Ctrl + F5` 후 새로 스캔합니다.

## 기존 설정 보존

업데이트 ZIP에는 `docs/config.js`와 `.github/`가 없습니다. 기존 Supabase 연결값, 개인 접속 토큰, GitHub 비밀값과 Actions 워크플로는 유지됩니다.

## 정상 확인

- 상단 엔진 버전 `v2.3.0`
- 최종 후보 카드의 동적 호가 관찰시간 약 60초 또는 90초
- 호가 25회 이상, 동시간대 체결 20건 이상, 구간 3/3
- 저유동 종목은 체결 20건을 채우지 못해 `동적 표본 부족`이 나올 수 있으며 이는 정상적인 안전 차단입니다.

공개 호가로 주문 주체나 숨은 주문을 볼 수 없으므로 관찰시간을 늘려도 스푸핑·아이스버그는 확정이 아닌 반복 패턴 기반 의심 판정입니다.

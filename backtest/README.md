# v2.5.0 백테스트 — 캔들 신호 측정 도구

배포 엔진의 `analyzePeriod`, `finalizeCandidate`, `buildTradePlan`을 그대로
호출해 과거 **마감 캔들**로 BUY와 조건부 WAIT 진입을 별도로 평가합니다.

## 이 도구가 측정하는 것

- 5분·15분·4시간·일봉 추세와 모멘텀
- 24시간 거래대금·등락·고저 범위의 시점별 재구성
- 즉시 BUY 신호의 단기 목표·손절 결과
- `관찰·눌림 대기`의 가격대 도달 → 15분 EMA21 회복 → 다음 봉 진입 결과
- 수수료, 진입·청산 슬리피지, 위험예산 기반 실제 투입비중
- 승률과 95% Wilson 구간, 기대값, Profit Factor, MDD

## 측정하지 못하는 것

공개 REST API에는 과거 호가창 전체와 체결 스트림이 없습니다. 따라서 스푸핑성
취소, 매도 재보충·흡수, 지지 붕괴, 돌파 후 지지 전환은 과거 캔들로 검증할 수
없습니다. 백테스트에서는 이 부분을 중립으로 둡니다.

중립 결과는 라이브 성과의 상한이나 하한이 아닙니다. 동적 필터가 손실 신호를
제거하면 성과가 좋아질 수 있고 이익 신호를 제거하면 나빠질 수 있습니다.
동적 호가 로직은 60~90초 원본 표본과 당시 추천을 저장한 뒤 전진 페이퍼
평가해야 합니다.

현재 상장 종목의 과거 캔들만 받을 수 있으므로 상장폐지 종목과 과거 유의종목을
완전히 복원하지 못합니다. 따라서 생존편향이 남습니다. 여러 파일을 함께 넣은
결과는 동시 자본배분이 없는 pooled 신호 통계이며 진짜 포트폴리오 자본곡선이
아닙니다.

## 미래참조 방지

- 결정시각까지 완전히 마감된 봉만 엔진에 전달
- 4시간봉·일봉도 `시작시각 + 주기 <= 결정시각`일 때만 사용
- 신호 다음 15분봉 시가에 설정된 진입 슬리피지를 더해 체결
- 목표·손절 도달은 진입 이후 봉에서만 검사
- 같은 봉에서 목표와 손절이 모두 닿으면 손절 우선
- 훈련·검증·테스트 경계 밖의 청산 결과는 사용하지 않음
- 일봉 50개가 완성된 뒤부터만 평가

## GitHub 웹에서 실행

1. 저장소 `Actions`를 누릅니다.
2. 왼쪽 `Backtest Market Scanner`를 누릅니다.
3. `Run workflow`를 누릅니다.
4. 거래소와 종목을 입력합니다.
   - 업비트: `KRW-BTC,KRW-ETH`
   - 바이낸스: `BTCUSDT,ETHUSDT`
5. 기간은 최소 55일, 권장 180일 이상으로 입력합니다.
6. 먼저 `sweep=false`로 기본값을 확인합니다.
7. 연구가 필요할 때만 `sweep=true`로 실행합니다.
8. 완료 후 실행 화면의 요약 또는 `Artifacts`에서 결과를 확인합니다.

공개 API만 사용하므로 GitHub 비밀값은 필요하지 않습니다. 한 번에 최대 10개
종목으로 제한해 거래소 부하와 Actions 시간을 통제합니다.

## 60/20/20 연구 모드

`sweep=true`는 공통 평가기간을 다음처럼 고정합니다.

- TRAIN 60%: 파라미터 조합 선택에만 사용
- VALIDATION 20%: 선택된 한 조합의 일반화 여부 확인
- HOLDOUT TEST 20%: 마지막 단 한 번의 미사용 구간 확인

검증·테스트 결과로 다시 조합을 선택하지 않습니다. 거래 40건은 자동 필터의
최소치일 뿐 통계적 충분성을 보장하지 않습니다. 종목·상승장·하락장·횡보장을
바꿔 반복하고, 운영 기본값은 전진 페이퍼 결과까지 유지하십시오.

## 로컬 Deno 사용법(선택)

```bash
deno test supabase/functions/market-scanner/engine.test.ts backtest/*.test.ts
deno run -A backtest/fetch-history.ts upbit KRW-BTC 180
deno run --allow-read=backtest/data backtest/run.ts backtest/data/upbit-KRW-BTC.json
deno run --allow-read=backtest/data backtest/run.ts --sweep backtest/data/*.json
```

# Trading-booooo v2.6.0 업데이트

## 결론

v2.6.0은 “현재 조건을 설명하는 스캐너”에서 “과거 신호와 실제 이후 가격을 자동 대조하고, 검증을 통과한 경우에만 스스로 파라미터를 교정하는 스캐너”로 확장한 버전입니다.

## 핵심 수정

1. 가까운 저항만으로 중기 후보까지 일괄 평가하던 문제 수정
   - 단기 일괄청산과 60/40 분할청산 분리
   - 단기·중기·혼합 손익비 동시 표시
2. 손절 구조 보완
   - ATR15 0.35배, 스프레드 3배, 2틱 중 최대값을 지지 하단 완충폭으로 사용
3. 추천 상승률 정교화
   - 목표가 복사 방식 폐기
   - 점수구간별 실제 MFE·목표 선도달률 기반 축소·보정
   - 보수/중심/낙관 가격 범위와 교정표본 표시
4. WAIT 백테스트 오류 수정
   - 현재가 계획값을 재사용하지 않고 실제 눌림 진입가·목표·손절로 재계산
5. 백테스트 확장
   - BUY, WAIT, 거절 후보를 모두 평가
   - 같은 봉 목표·손절은 손절 우선
   - MFE·MAE·예측오차·Brier Score 추가
6. 자동 교정
   - TRAIN/VALIDATION/HOLDOUT 완전 분리
   - 승격 가드 실패 시 기존값 유지
   - 매주 자동 실행 및 승격 프로필 자동 재배포
7. 운영 누락 수정
   - 실제로 없던 백테스트 Workflow 추가
   - Supabase Workflow 자기 경로 수정
   - `.gitignore`, `.nojekyll`, 환경변수 예시 추가

## 자동 교정하지 않는 것

- HOLDOUT 결과를 보고 파라미터를 다시 고르지 않음
- 검증 실패 프로필을 운영에 적용하지 않음
- 과거에 없는 실시간 호가·체결을 임의로 만들어 백테스트하지 않음
- 샘플 부족 상태에서 확률을 측정값처럼 표시하지 않음

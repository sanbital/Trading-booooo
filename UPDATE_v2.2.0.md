# v2.2.0 업데이트 — GitHub 웹에서만 적용

이 ZIP은 기존 v2.1.0 설치본을 v2.2.0으로 덮어쓰는 업데이트 파일입니다. VS Code, Git, 터미널은 필요하지 않습니다.

## 포함된 변경

- 현재 매수 후보의 `진입 시 예상 매도가`
- 관찰·눌림대기 후보의 조건부 매수가·예상 매도가·손절가·진입/매도 시나리오
- 업비트 KRW / 바이낸스 USDT 현물 별도 탭
- Binance 공개 REST·WebSocket 기반 기간·동적 호가 분석

## 적용 순서

1. `Trading-booooo-v2.2.0-GitHub-Update.zip`을 다운로드해 압축을 풉니다.
2. GitHub의 `Trading-booooo` 저장소 → `Code` → `Add file` → `Upload files`로 이동합니다.
3. 바깥쪽 폴더가 아니라 압축을 푼 폴더 **안의 모든 항목**을 업로드 영역에 끌어놓습니다.
4. Commit message에 `Update scanner scenarios and Binance v2.2.0`을 입력합니다.
5. `Commit changes`를 누릅니다.
6. `Actions`에서 `Deploy Supabase Market Scanner`와 `pages build and deployment`가 모두 초록색인지 확인합니다.
7. 기존 개인 접속 URL을 열고 `Ctrl + F5`를 누릅니다.

## 보존되는 파일

업데이트 ZIP에는 아래 파일이 의도적으로 없습니다.

- `docs/config.js`
- `.github/` 전체

따라서 기존 Supabase 주소·Publishable Key·개인 토큰·Actions 비밀값과 배포 워크플로는 덮어쓰지 않습니다.

## 정상 확인

1. 상단 버전이 `v2.2.0`인지 확인합니다.
2. `업비트 현물`과 `바이낸스 현물` 두 탭이 보이는지 확인합니다.
3. 업비트 탭은 자본 단위가 원, 바이낸스 탭은 USDT인지 확인합니다.
4. 현재 매수 후보 카드에 `진입 시 예상 매도가`가 표시되는지 확인합니다.
5. `관찰·눌림 대기` 카드에는 조건부 매수가·예상 매도가·손절가와 진입/매도 문장이 모두 있어야 합니다.
6. 가격 시나리오를 안전하게 만들 수 없는 후보는 `관찰·조건 회복 대기`로 표시되고 가격이 숨겨지는 것이 정상입니다.

## 주의

- 두 탭의 점수와 순위는 각 거래소 안에서만 비교합니다.
- 바이낸스는 USDT 현물이며 선물·마진·레버리지 토큰 분석이 아닙니다.
- Binance API Key는 필요하지 않으며 자동 주문은 없습니다.
- 예상 매도가는 보장 가격이 아니라 현재 지지·저항·ATR·수수료·슬리피지를 반영한 조건부 추정입니다.

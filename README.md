# Trading-booooo Market Scanner v2.2.0

업비트 KRW 현물과 바이낸스 USDT 현물을 별도 탭에서 전수 점검하고, 기간 추세와 최신 호가·체결을 함께 통과한 종목만 현재 매수 후보로 제시하는 개인용 읽기 전용 분석 도구입니다.

자동 주문, 거래소 계정·잔고 조회, 거래소 API Key 사용은 포함하지 않습니다. 두 거래소 모두 공개 현물 시세만 읽습니다.

## v2.2 핵심 동작

1. 업비트 KRW 또는 바이낸스 USDT 탭을 선택해 해당 현물시장 전체의 현재가·24시간 거래대금·체결 최신성을 1차 점검합니다.
2. 유의·주의·저유동·장기 미체결 종목을 제외합니다.
3. 안전필터를 통과한 **모든 종목**의 15분봉 96개(약 24시간)를 먼저 조회해 기간 추세를 점검합니다.
4. 15분 기간추세·유동성·모멘텀·당일 위치를 합산해 정밀분석 30개 후보를 구성하고 다음 시간축을 추가 조회합니다.
   - 5분봉 72개: 최근 약 6시간
   - 4시간봉 90개: 최근 약 15일
   - 일봉 60개: 최근 약 60일
5. 기간점수 상위 8개는 선택 거래소의 공개 WebSocket으로 기본 18초간 호가·체결을 동시에 관찰합니다.
6. 가격대별 잔량 변화와 같은 시간·가격의 실제 체결을 교차검증해 비체결성 대형벽 취소, 매도벽 재보충·흡수, 지지 붕괴, 돌파 후 지지 전환을 판정합니다.
7. WebSocket 연결 실패·관찰 10초 미만·서로 다른 호가 8회 미만·동시간대 체결 8건 미만이면 동적 표본 부족으로 BUY를 금지합니다.
8. 모든 강제 게이트를 통과한 최대 3개만 `현재 매수 후보`로 표시합니다. 통과 종목이 없으면 `현재 매수 추천 없음`이 정상 결과입니다.
9. 현재 매수 후보에는 수수료와 출구 슬리피지를 반영한 `진입 시 예상 매도가`를 표시합니다.
10. `관찰·눌림 대기`는 매수구간·예상 매도가·손절가·진입조건·매도조건이 모두 계산될 때만 표시합니다. 시나리오를 만들 수 없는 WAIT는 `관찰·조건 회복 대기`로 분리합니다.

## 추천 결과

- 현재 매수 후보와 종합점수·신뢰도
- 매수 검토 구간
- 진입 시 예상 매도가(수수료·출구 슬리피지 반영)
- 단기 목표가와 중기 목표가
- 손절가격과 비용 반영 예상 손실률
- 위험예산 기반 추천 투입금
- 단기·중기·중장기 보유 분류와 조건부 추세 유지기간
- 추세 무효화 조건
- 5분·15분·4시간·일봉 지표
- 최신 스프레드·체결 압력과 정적 호가 불균형 참고값
- 동적 호가 관찰시간·서로 다른 호가 표본·동시간대 체결 표본·데이터 품질
- 가짜 매수벽 취소 의심·매도 재보충/흡수·지지 붕괴·돌파 후 지지 전환 판정
- 반복 유지된 매도벽을 반영한 보수적 단기 목표가 상한
- 좋은 추세지만 현재가 진입이 부적합한 WAIT 후보의 조건부 대기 매수구간·예상 매도가·손절가
- 눌림 도달 → 15분봉 회복 확인 → 진입 검토 → 지정가 분할매도 또는 조기 종료로 이어지는 시나리오
- 가격 도달 후 15분봉·호가·체결·손익비를 다시 확인하는 예약매수 전 조건
- GPT/Claude에 그대로 붙여넣을 수 있는 Markdown 심층분석 리포트

## 보안 방식

로그인 화면은 없습니다. GitHub Pages 주소 뒤의 URL fragment에 개인 토큰을 넣습니다.

```text
https://YOURNAME.github.io/Trading-booooo/#access=YOUR_PRIVATE_TOKEN
```

fragment는 GitHub 서버 요청에 포함되지 않으며, 브라우저 앱이 Supabase Edge Function 호출 헤더로 전달합니다. Edge Function은 Supabase Secret의 `SCAN_ACCESS_TOKEN`과 일치할 때만 스캔합니다.

> GitHub Pages 주소 자체는 공개적으로 발견될 수 있습니다. 개인 토큰이 없으면 분석 함수는 실행되지 않지만, 토큰이 포함된 전체 URL을 공유하거나 캡처하지 마세요.

## 설치 방식

VS Code, PowerShell, Git, npm, Supabase CLI가 필요 없습니다. GitHub 저장소 비밀값 3개를 등록하고 파일을 웹으로 업로드하면 `.github/workflows/deploy-supabase.yml`이 Edge Function 비밀값 설정과 배포를 자동으로 처리합니다. 프론트엔드 설정도 GitHub의 파일 편집 화면에서 두 값만 바꿉니다.

## 투자 유의사항

본 도구는 공개 시세 기반 조건부 분석만 제공합니다. 목표가·손절가·보유기간은 미래를 확정하는 값이 아니며, 뉴스·공시·상장폐지·급격한 유동성 공백과 실제 체결 슬리피지를 완전히 반영할 수 없습니다. 수익을 보장하지 않으며 최종 판단과 책임은 사용자에게 있습니다.

조건부 대기 매수구간은 자동 주문 가격이 아닙니다. 가격이 도달한 뒤에도 지정된 재확인 조건과 최신 스캔의 강제 게이트를 통과해야 유효합니다.

# v2.4.0 업데이트 — 업비트·바이낸스 통합 Top 4

이 문서는 현재 GitHub 저장소를 v2.4.0으로 덮어쓰는 가장 짧은 순서입니다. VS Code, PowerShell, Git 명령은 사용하지 않습니다.

## 바뀌는 내용

- 기본 탭에서 업비트 KRW와 바이낸스 USDT 현물을 동시에 스캔합니다.
- 같은 코인이 양쪽 거래소에 잡혀도 통합 Top 4의 한 자리만 사용합니다.
- 양쪽이 모두 BUY이면 제한적인 확인 가산점을 줍니다.
- 반대 거래소에서 회피·스푸핑성 취소·매도흡수·지지붕괴가 확인되면 현재가 BUY를 대기로 낮춥니다.
- 안전 후보가 4개보다 적으면 위험 후보로 빈자리를 채우지 않습니다.
- 기존 `업비트 현물`, `바이낸스 현물` 개별 탭은 그대로 남습니다.

## GitHub 웹에서 업데이트

1. `Trading-booooo-v2.4.0-GitHub-Update.zip`을 다운로드하고 Windows에서 압축을 풉니다.
2. GitHub의 `Trading-booooo` 저장소에서 `Code` → `Add file` → `Upload files`를 누릅니다.
3. 압축을 푼 폴더 **안의 모든 항목**을 업로드 영역으로 끌어놓습니다.
4. `supabase/functions/market-scanner/combined.ts`와 `combined.test.ts`가 업로드 목록에 포함됐는지 확인합니다.
5. Commit message에 `Add combined Top 4 v2.4.0`을 입력하고 `Commit changes`를 누릅니다.
6. `Actions` → `Deploy Supabase Market Scanner`에서 새 실행의 `Deploy market-scanner`가 초록색인지 확인합니다.
7. `Actions` → `pages-build-deployment`도 초록색이 될 때까지 기다립니다.
8. 개인 접속 URL을 열고 `Ctrl + F5`로 한 번 새로고침합니다.

업데이트 ZIP에는 기존 Supabase 주소와 Publishable Key를 보존하도록 `docs/config.js`를 넣지 않습니다. 기존 GitHub의 `docs/config.js`는 수정할 필요가 없습니다.

## 화면 확인

1. 왼쪽 위 버전이 `v2.4.0`인지 확인합니다.
2. `통합 Top 4`, `업비트 현물`, `바이낸스 현물` 탭 3개가 보이는지 확인합니다.
3. 통합 탭에서 원화 자본과 USDT 자본을 확인하고 `두 거래소 동시 스캔`을 누릅니다.
4. 스캔은 양쪽에서 병렬로 진행되지만 각 거래소의 60~90초 동적 호가 관찰과 API 호출이 있어 최대 약 260초가 걸릴 수 있습니다.
5. 후보 카드에서 대표 거래소, 통합점수, 거래소별 판정이 표시되는지 확인합니다.
6. 같은 기초자산이 통합 Top 4에 중복으로 나오지 않는지 확인합니다.

한 거래소만 일시 실패해도 전체 요청을 버리지 않고 성공한 거래소 결과를 표시합니다. 이때 화면과 복사 리포트에 실패한 거래소 오류가 남습니다.

## 중요한 판정 원칙

통합점수는 두 거래소의 가격이나 거래대금을 합산한 값이 아닙니다. 각 거래소 엔진의 동일한 0~100점 척도를 바탕으로 중복 제거와 교차 확인만 반영한 정렬 점수입니다. KRW와 USDT 값은 서로 환산하지 않고 원 단위 그대로 표시합니다.

이 도구는 자동 주문을 하지 않습니다. 조건부 매수가는 가격 도달 즉시 주문하라는 뜻이 아니라, 도달 후 15분봉·동적 호가·체결·손익비를 다시 확인하기 위한 관찰 가격입니다.
